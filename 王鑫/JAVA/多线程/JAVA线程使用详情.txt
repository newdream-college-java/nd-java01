 多线程
一、为什么需要多线程
      多线程可以同时进行多个操作,从而提高软件的运行效率.。
二、什么是进程、线程
      1.什么是进程--内存资源
            进程就是计算机运行的一组应用程序，而每个进程计算机都会为其分配相应的内存等资源
      2.什么是线程--临界资源
            当计算机运行一个程序，而这个程序内部至少由一个进程组成，例如：java中的main方法也是一个(main)线程。
      3.进程与线程之间的关系
            一个程序包含多个进程，一个进程包含多个线程(线程又包含多个携程)
三、并行与并发
      1.并发
            指多个程序在某一时间段运行，CPU会为其分配相应的资源空间，但处理器只支持单进程，就会将进程进行轮流操作运行。
      2.并行
            指多个程序在某一时间段运行，CPU会为其分配相应的资源空间，但处理器可支持多进程，此时多个程序同时运行(多核的CPU)。
四、如何使用线程
      方式1：继承Thread类
            第一步：创建一个类去继承Thread类
            第二步：内部重写run方法
            第三步：将需要执行的代码写入run方法内部
            第四部：在主类里面创建继承Thread方法的类对象
            第五步：对象调用start()方法启动线程(其运行线程个数为2，main--默认 和 Thread 对象线程)
      方式二：实现Runnable接口
            第一步：创建一个类实现Runnable接口
            第二步：同上方法二
            第三步：同上方法三
            第四步：主类里面创建Runnable接口对象
            第五步：创建Thread对象，将Runnable对象传入Thread对象中
            第六步：同上方法五 
            补充：也可以直接创建Runnable对象直接调用run方法执行，同样可以达到启动线程的效果(不推荐)
五、两种创建线程的区别
      1.继承Thread
            好处：可以直接使用Thread类中的方法
            坏处：Java中只能单继承无法多继承
      2.实现Runnable接口
            好处：该方法是一接口形式存在，即使你该类继承了其他类，但接口可以多实现
            弊端：不能直接使用Thread方法，需要得到对象才能调用，代码复杂。
六、线程类方法
      currentThread()获取当前线程对象		例如：线程对象.currentThread().getName()
      getName()和setName()获取当前线程名和设置当前线程名
      setDaemon(Boolean)设置一个线程为守护线程
      sleep(毫秒)将程序休眠若干秒再执行
      join()将线程暂停,等指定线程结束后再运行
七、什么是线程安全
      如果有多个线程同时运行，所产生的数据与单线程运行结果不一样，就会造成线程的不安全
八、线程不安全的三个关键点
      1.资源共享：同一资源被多个线程使用，造成数据与单线程不一致
      2.多线程：多个线程相互争抢资源造成数据的不一致
      3.并发问题：一个结果被多个线程同时使用
十、如何解决多线程并发问题
      在并发编程中，线程安全问题的本质其实就是 原子性、有序性、可见性。
      1、文件锁
            如果对该表的更新或插入的操作，都会经过一个统一的文件，这种方式是可以解决的多进程并发的问题；
      2、序列化接口（对象序列化）
            所有php里面的值都可以使用函数serialize()来返回一个包含字节流的字符串来表示。unserialize()函数能够重新把字符串变回php原来的值。 序列化一            个对象将会保存对象的所有变量，但是不会保存对象的方法，只会保存类的名字。
      3、事务隔离级别
      数据库事务的隔离级别有4个，由低到高依次为Read uncommitted、Read committed、Repeatable read、Serializable，这四个级别可以逐个解决脏读      、不可重复读、幻读这几类问题。
